# Product Areas

This directory records user-facing behavior. It should describe what the product currently does, not roadmap wishes.

## Documents

- `pet-window-experience.md`: floating pet window behavior.
- `task-card-interaction.md`: task card content, actions, dismissal, reply, and approval.
- `personalization.md`: pet, bubble, sound, theme, and opacity settings.
- `notification-and-approval.md`: attention states and approval flow.

## Evidence Sources

- `frontend/App.svelte` for settings UI.
- `frontend/PetApp.svelte` for pet window behavior.
- `frontend/lib/activity.ts` and tests for task card rules.
- `frontend/lib/sound.ts` and tests for notification behavior.
