# Runbooks

Runbooks are repeatable debugging paths. They should start from observable symptoms and avoid assuming a root cause too early.

## Available Runbooks

- `hook-not-working.md`
- `reply-not-shown-in-app.md`
- `window-overflows-screen.md`
- `task-list-collapsed-or-clipped.md`

## Writing Rule

Use evidence first: logs, generated config, local state, failing test, screenshot, or reproduction steps. If the root cause is unknown, write it as unknown.
